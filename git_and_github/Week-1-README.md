## ⭐ The 10 Commits You'll Make

Your goal is to create a real project with **exactly 10 meaningful commits**:

```
📌 COMMIT 1: Initial commit with README and gitignore
   └─ Create project structure and documentation

📌 COMMIT 2: Add HTML structure and title
   └─ Create basic HTML file

📌 COMMIT 3: Add CSS styling and gradient background
   └─ Style the project beautifully

📌 COMMIT 4: Add JavaScript with console messages
   └─ Add interactivity

📌 COMMIT 5: Add navigation, sections, and footer
   └─ Expand HTML content

📌 COMMIT 6: Add improved styling (on feature branch)
   └─ Create and switch to feature/improve-styling branch

📌 COMMIT 7: Add smooth scrolling features
   └─ Continue on feature branch

📌 COMMIT 8: Merge feature branch
   └─ Merge feature/improve-styling back to main

📌 COMMIT 9: Add contact form section with styling
   └─ Back on main, add new features

📌 COMMIT 10: Complete documentation and final polish
   └─ Update README and finalize project
```

## 🎯 Quick Start (TL;DR)

If you're experienced, here's the fast path:

```bash
# 1. Setup
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 2. Create project
mkdir git-practice-seedwoc
cd git-practice-seedwoc
git init

# 3. Make 10 commits
# (Follow the commits tracker above)

# 4. Create GitHub repo at github.com/new
# Name: git-practice-seedwoc

# 5. Push to GitHub
git remote add origin https://github.com/YOUR-USERNAME/git-practice-seedwoc.git
git branch -M main
git push -u origin main
```

## 💡 Key Concepts

### The 3 States in Git
```
Working Directory (files you edit)
         ↓ git add
Staging Area (selected files)
         ↓ git commit
Repository (saved versions/history)
```

### The 10 Commits Strategy
```
Commits 1-5:  Build basic project (HTML, CSS, JS, structure)
Commits 6-8:  Learn branching (create feature branch, merge)
Commits 9-10: Add more features and documentation
```

### The Project Journey
```
START → Init → Commit 1 → Commit 2 → ... → Commit 10 → GitHub → SUCCESS! 🎉
```

## 🔄 The Git Workflow You'll Learn

```bash
# 1. Create files
# 2. Check status
git status

# 3. Stage changes
git add .

# 4. Commit
git commit -m "Meaningful message"

# 5. Repeat steps 1-4 nine more times!

# 6. View history
git log --oneline

# 7. Push to GitHub
git push origin main
```

## 📊 Quick Reference

### Essential Git Commands (10)
```bash
git init                          # Start tracking
git add <file>                   # Stage file
git commit -m "message"          # Save snapshot
git log --oneline                # View history
git branch <name>                # Create branch
git checkout -b <name>           # Create & switch
git merge <branch>               # Combine branches
git push origin main             # Upload to GitHub
git remote add origin <url>      # Connect to GitHub
git branch -d <name>             # Delete branch
```

### Essential Bash Commands (10)
```bash
pwd                              # Current location
ls                               # List files
mkdir <name>                     # Create folder
cd <folder>                      # Enter folder
touch <file>                     # Create file
cat <file>                       # View file
echo "text" > <file>            # Write to file
git --version                    # Check Git
clear                            # Clear screen
whoami                           # Your username
```
