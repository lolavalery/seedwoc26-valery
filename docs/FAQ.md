# FAQ - Frequently Asked Questions

## Getting Started

### Q: I'm completely new to programming. Where should I start?
**A:** Start with Chapter 1 (Version Control) - it's essential foundation. Then choose Chapter 2 (JavaScript) or Chapter 4 (Python) based on your interest. Both are beginner-friendly.

### Q: How long will it take to complete the entire program?
**A:** It depends on your pace. Each chapter takes 2-5 weeks. Completing all 17 chapters typically takes 12-18 months of consistent study (15-20 hours/week). You don't need to complete all chapters - create your own learning path.

### Q: Do I need to be good at math?
**A:** Not for the first few chapters. For machine learning and advanced AI chapters, some statistics and linear algebra helps, but you can learn it as you go.

## Technical Questions

### Q: Which programming language should I learn first?
**A:** JavaScript (Chapter 2) if you want web development, Python (Chapter 4) if you want data science/AI, Dart (Chapter 5) if you want mobile development.

### Q: Do I need an expensive computer?
**A:** No. All tools are free and open-source. Any computer from the last 5 years will work fine. Even a Raspberry Pi can run most exercises.

### Q: What if I already know some topics?
**A:** You can skip chapters! Read through the README for each chapter and jump to what's new for you.

### Q: Can I work offline?
**A:** Most material yes, but some chapters (especially web development) benefit from access to documentation. Download what you need beforehand.

## Git & GitHub

### Q: Do I need GitHub?
**A:** Not required to learn, but highly recommended. It's how professionals collaborate and showcase work. Free account available at github.com.

### Q: What if I mess up my Git repository?
**A:** Git is very forgiving. You can almost always recover deleted commits with `git reflog`. Worst case, delete the repository and start fresh.

### Q: How do I contribute my solutions?
**A:** Follow the structure: `assignments/{github_username}/{chapter}/{module}/{part}/`. Submit a pull request.

## Assignments & Submissions

### Q: Are the solutions available?
**A:** Lecture notes and hints are provided. Full solutions are available in a separate instructors-only repository. Use them to check after you've tried.

### Q: How are submitted assignments reviewed?
**A:** All PRs are reviewed for code quality, correctness, and following best practices. You'll receive feedback within 48 hours.

### Q: Can I submit incomplete work?
**A:** You can create a draft PR, but completion is required for final acceptance. Use issues to ask for help if stuck.

### Q: Do I get feedback on my code?
**A:** Yes! Reviewers provide constructive feedback. Use it to learn better practices.

## Learning & Difficulty

### Q: What if I'm stuck on an exercise?
**A:** 1) Read the hints, 2) Review the lecture notes, 3) Search online, 4) Post on discussions, 5) Create an issue asking for help.

### Q: Is the difficulty progression good?
**A:** Yes. Each part builds on previous knowledge. Start with basics, progress gradually.

### Q: Can I jump between chapters?
**A:** Some chapters are independent, but many build on earlier ones. Follow the prerequisites listed in each chapter README.

### Q: How do I know if I understand something?
**A:** You can explain it to someone else, you can teach it in your own words, and you can apply it to new problems. If you can do these, you understand it.

## Projects & Practice

### Q: Are the final projects mandatory?
**A:** They're highly recommended but not strictly required. They help consolidate learning and build a portfolio.

### Q: Can I modify the suggested projects?
**A:** Absolutely! Make them your own. Solve problems you care about using the skills learned.

### Q: How do I build a portfolio?
**A:** Push all your work to GitHub. Include READMEs explaining your projects. This is your portfolio.

### Q: Can I contribute my own exercises?
**A:** Yes! Submit them following the guidelines in CONTRIBUTING.md. We'd love community contributions.

## Career & Certifications

### Q: Will this lead to a job?
**A:** These skills are highly employable. But you'll need to:
1. Build real projects
2. Complete portfolio
3. Apply to jobs
4. Network with others

The program teaches you skills; you build the career.

### Q: Is there a certification?
**A:** Completion badges are available for finishing tiers of chapters. These show employers you have real skills.

### Q: Should I do all chapters or specialize?
**A:** Specialize unless you're early in your career. Pick a path that interests you and go deep.
